1. 인터페이스의 역할
- DAO등의 메서드 추상 선언을 담아 어떤 이름, 파라미터, 리턴값을 가지는 지 알려주는 파일

2. 인터페이스에 들어가는 2가지
- 상수형 변수
- 추상메서드

3. 인터페이스와 추상클래스의 공통점/차이점
- 공통점:
1) 상속할 수 있다
2) 추상메서드를 가진다
3) 객체 생성 불가 (new로 못만듦!)
- 차이점:
1) 인터페이스는 다중상속이 가능하다 (인터페이스끼리는 extends-상속 사용, 인터페이스로 클래스 만들때는 implements)
2) 인터페이스는 클래스 내 상수를 제외하고 변수선언이 불가능하다
3) 인터페이스는 일반변수, 일반메서드 선언 불가능

4. 추상클래스와 일반클래스의 공통점/차이점
- 공통점: 일반변수, 일반메서드 선언 가능
- 차이점: 
1) 추상클래스는 객체생성이 불가능하다 
2) 추상클래스는 추상메서드를 가진다 (일반클래스는 추상메서드 없어야함)

5. 추상클래스에는 일반변수가 올 수 있나요?
- Yes

6. 인터페이스에는 일반변수가 올 수 있나요?
- No

7. 추상 클래스와 일반클래스의 구분 기준
- 추상메서드를 가지고 있는지 여부 (추상 메서드가 하나라도 있으면 추상클래스)

8. 추상클래스는 객체생성할 수 있나요.
- No (추상클래스를 상속하는 일반클래스를 만들어서 객체 생성해야한다)

9. 인터페이스는 객체생성할 수 있나요.
- No

10. ApplePhone p = new Apple13(); 가능한가?
- 가능하다
- 추상클래스(ApplePhone)를 상속하는 일반클래스(Apple13)
- 자식 일반클래스의 객체를 만들어서 형변환하여 부모클래스의 객체(p)에 대입한것
