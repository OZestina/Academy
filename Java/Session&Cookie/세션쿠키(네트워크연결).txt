0. 네트워크 연결에서 비연결성이란?
- http는 연결을 유지하지 않은 채 (클라이언트와 서버 간의) 응답과 요청을 처리함
- 요청에 대한 응답을 처리하면 연결을 끊는다

1. 네트워크 연결에서 무상태성이란?
- 비연결성으로 인해 이전에 저장했던 상태값을 계속 유지할 수 없음
- 비연결성으로 인해 http에는 이전의 정보나 현재 통신의 상태가 남아있지 않음

2. 네트워크 연결에서 상태를 저장하기 위한 두 가지 기술은?
- 세션과 쿠키
- 세션은 서버의 램에 저장되는 상태 정보
- 쿠키는 클라이언트의 브라우저에 저장되는 상태 정보

3. 세션과 쿠키의 차이점은?
- 세션: 서버의 램에 저장, 하나씩 저장했다가 하나씩 꺼내쓸 수 있음. 저장하는 타입이 아무거나 괜찮다
- 크키: 클라이언트의 브라우저에 저장, 하나씩 저장했다가 전체를 서버에서 가지고 와서 꺼내써야 한다. 모두 다 스트링만 가능하다

4. 일반적으로 세션이란?
- 네트워크 연결에서 클라이언트가 서버에 연결하는 하나의 단위
- 연결이 완벽히 끊기기 전까지의 연결의 단위

5. 웹 프로그램에서 세션이란?
- 브라우저가 닫히기 전까지의 서버와의 연결 (자동으로 세션 종료)
- 로그아웃하는 경우(수동으로 세션 종료)
